# 네이버 광고 데이터 파이프라인

네이버 SA(쇼핑검색·파워링크)와 GFA(애드부스트·소재보고서) 광고 데이터를 처리해 대시보드용 CSV를 생성하는 자동화 도구입니다.

---

## 폴더 구조

```
naver_ad_tool/
├── raw/                          ← 매달 교체하는 원본 파일
│   ├── 노출클릭_N월.csv           ← SA 노출/클릭/비용 (API 또는 수동 다운로드)
│   ├── 구매완료_N월.csv           ← SA 전환 데이터 (API 또는 수동 다운로드)
│   └── GFA/
│       ├── 애드부스트_N월.csv
│       └── 소재보고서_N월.csv
│
├── 기준데이터/                    ← 거의 바뀌지 않는 기준 파일
│   ├── 광고 캠페인 리스트.csv      ← 쇼핑검색 소재ID → 상품ID 매핑
│   ├── 광고 캠페인 리스트_파워링크.csv  ← (현재 미사용, 참고용)
│   ├── 스토어 등록 상품 리스트.csv  ← 상품ID → 모델명/카테고리 매핑
│   └── 모델명 리스트.csv           ← 파이프라인이 자동 생성·누적 (수동 수정 가능)
│
├── output/                        ← 결과 파일 (자동 저장)
│   ├── sa_data.csv
│   └── gfa_data.csv
│
├── naver_ad_pipeline.py           ← SA 파이프라인 메인
├── gfa_ad_pipeline.py             ← GFA 파이프라인 메인
├── naver_api_download.py          ← SA raw 파일 API 자동 다운로드
├── run_manual.bat                 ← 통합 실행 (파이프라인 + GitHub 업로드)
├── run.bat                        ← API 다운로드 → 파이프라인 → GitHub 자동화
└── .env                           ← API 키 (Git 미포함)
```

---

## 실행 방법

### A. 수동 raw 파일이 있을 때 (매달 기본 작업)

1. 네이버 광고센터에서 해당 월 raw 파일 다운로드
   - SA: 노출클릭_N월.csv, 구매완료_N월.csv → `raw/` 폴더
   - GFA: 애드부스트_N월.csv, 소재보고서_N월.csv → `raw/GFA/` 폴더
2. `run_manual.bat` 실행

### B. API 자동 다운로드 (SA만 해당)

1. `run.bat` 실행 (API로 raw 파일 생성 후 파이프라인 자동 실행)
2. 또는 `naver_api_download.py` 단독 실행 후 `run_manual.bat` 실행

> **주의:** API는 당일 실시간 데이터를 제외하고 어제까지만 조회합니다 (광고센터 수동 방식과 동일).

## 모델명 리스트 시스템

### 개념

`기준데이터/모델명 리스트.csv`는 파이프라인이 자동으로 생성·누적하는 **모델명 가이드북**입니다.  
수동으로 한 번 교정해두면 이후 실행에서도 계속 적용됩니다.

### 컬럼 설명

| 컬럼 | 설명 |
|---|---|
| 자동추출_모델명 | 파이프라인이 자동으로 추출한 모델명 (키) |
| 상품ID | 네이버 스마트스토어 상품번호 (상품 조회용) |
| 캠페인명 | 해당 모델이 속한 캠페인명 (확인용) |
| 자동추출_카테고리 | 파이프라인이 자동으로 분류한 카테고리 |
| 수동 적용_모델명 | **직접 입력** — 자동추출이 틀렸을 때 올바른 모델명 입력 |
| 수동 적용_카테고리 | **직접 입력** — 카테고리 교정 시 입력 |

### 수동 적용 방법

1. `모델명 리스트.csv` 열기
2. 잘못된 행의 `수동 적용_모델명` 또는 `수동 적용_카테고리` 컬럼에 올바른 값 입력
3. 저장 후 파이프라인 재실행
4. 이후 실행에서도 수동 적용 값이 자동으로 우선 적용됨

> `수동 적용_모델명`만 입력하면 모델명만 교정, `수동 적용_카테고리`만 입력하면 카테고리만 교정.  
> 두 컬럼 모두 독립적으로 작동합니다.

### 동작 우선순위 (SA 쇼핑검색)

```
1순위: 스토어 등록 상품 리스트 매칭 (상품ID → 판매자상품코드)
2순위: 캠페인명에서 모델명 자동 추출 (예: 🔴TV65인치_KQ65SF8EAEXKR → KQ65SF8EAEXKR)
3순위: (미매칭)_캠페인명 으로 표시
```

수동 적용 값이 있으면 위 결과를 덮어씁니다.

## 카테고리 분류 기준

| 카테고리 | 해당 제품 |
|---|---|
| TV | QLED, OLED, Crystal UHD 등 삼성 TV |
| 에어컨 | 스탠드/벽걸이/창문형/멀티형 에어컨 |
| 시스템에어컨 | 시스템에어컨 (천장형 등) |
| 공기청정기 | 공기청정기 |
| 청소기 | 스틱/로봇/진공청소기 |
| 제습기 | 제습기 |
| SD | 사운드바, 블루투스 스피커 |
| 그 외 상품 | 위 분류 외 상품 |

## output 파일 구조

### sa_data.csv

| 컬럼 | 설명 |
|---|---|
| date | 연월 (YYYY-MM) |
| ad_type | 쇼핑검색 / 파워링크 |
| category | 카테고리 |
| model | 모델명 |
| spend | 광고비 (원, VAT 포함) |
| clicks | 클릭수 |
| impressions | 노출수 |
| conv | 구매완료 전환수 |
| revenue | 구매완료 전환매출액 (원) |

### gfa_data.csv

애드부스트(모델/상품 단위)와 소재보고서(캠페인 단위) 통합 파일.

## 주의사항

- **`기준데이터/` 파일 교체 시기:** 신규 캠페인이 생기거나 상품 구성이 바뀌면 교체
- **`(미매칭)` 모델명:** 소재가 삭제됐거나 캠페인 리스트에 없는 경우 발생 — `모델명 리스트.csv`에서 수동 적용으로 해결
- **CFX- 모델:** 스토어 상품 리스트에 판매자상품코드가 `CFX-`로 불완전 등록된 상품 7개 존재 — 스마트스토어에서 상품번호로 조회 후 코드 수정 권장
- **`.env` 파일:** API 키가 담겨 있어 Git에 포함되지 않음 — 분실 시 네이버 광고 API에서 재발급 필요
